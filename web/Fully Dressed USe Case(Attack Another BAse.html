Use Case6.0: Attack Another Base

Title: Attack Another Base
Primary Actor:Player
System under discussion:Strategy Game “Last Whisper”.
Level:Player Goal
Preconditions:The player is logged on, has started a new game or loaded a previous one
and is on his base map.
Postconditions:The Resources and army statistics are updated and maps details are reset
Success Guarantee:The Player has the ability to attach another base in the game.
Basic Flow:
  1. The Player goes to the map
  2. The Player selects a Base to attack.
  3. The Player selects The Attack option.
  4. The system calculates the total Attack value of the Player’s army as the offensive side and the
    total Defence value of the target base as the defensive side.
  5. The System determines the winner by comparing the Attack value of the offensive side and the
    Defence value of the defensive side.If the Attack value of the offensive side is at least 50% higher
    than the Defence value of the defensive side then the offensive side automatically wins.
  6. System gives a random part of the resources and of the remaining units of the losing side to
    the winning side.
  7. System shows the final battle screen.
  8. Player comes back to the updated map.
Alternative Flows:
  • 5.a:If the Defence value of the defensive side is at least 50% higher than the Attack value
    of the offensive side then the defensive side automatically wins.
  • 5.b:If the Attack value of the offensive side is equals to the Defence value of the defensive side
    then the System randomly determines the winner.
  • 6.1: If the defensive side has lost,the System sets the attacked base as destroyed.
Extensions:
  1. Player checks information about a base.
  2. Player comes back to the map.
