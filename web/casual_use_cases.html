Casual Use Cases

UC_1.0: Create a New Commander

Player wants to create a new commander. The player names their commander. The system will check if the name is valid. The player sets statistics for their commander: strength, intelligence and luck. The system checks if the total of these attributes add up to 30 (30 max).

UC_3.0: Create a New Building

The Player wants to create a new building. The Player will be presented with a list of buildings. The Player selects a building and the system checks to see if the player has the resources to build the selected building. If the player has the resources available they will be able to drag and drop their selected building to a chosen cell. The system checks if the target cell is free. If the cell is free then the selected building is constructed on the chosen cell. At any time prior to completing the building the Player can withdraw their building choice and resources will be reimbursed. The Use case ends when the building is complete.

UC_5.0: Create a New Unit

Player wants to delegate their army into units. The player will be shown a list of units. The player must selects a unit. The system will check if the player has the required building needed to house the unit. The player will select how many soldiers they want to delegate to that unit the system checks if the needed resources are there if they are the unit is created. The new unit will be created. The system will reduce the army quantity after the unit is created.
