Use Case Id: 6.0
Use Case Title: Attack Another Base
Primary Actor: Player
System under discussion: Turn Based Strategy Game "Last Whisper"
Level: Player Goal
Pre-conditions: The player is logged on, has started a new game or loaded a previous one and is on his base map.
Post-conditions: The armies and resources of the both sides are updated. If Player has won, the target base is destroyed and the map is updated.
Basic Flow:
  • 1.
    Player goes to the map.
  • 2.
    Player selects the target base.
  • 3.
    Player selects the attack option.
  • 4.
    Player chooses the number of each unit of his army he wants to take to attack the enemy’s base.
  • 5.
    System calculates the total attack value of the Player’s army as the offensive side and the total defence value of the target base as the defensive side.
  • 6.
    System determines the winner by comparing the attack value of the offensive side and the defence value of the defensive side.
    If the attack value of the offensive side is at least 50% higher than the defence value of the defensive side then the offensive side automatically wins.
  • 7.
    System gives a random part of the resources and a random part of the remaining units of the losing side to the winning side.
  • 8.
    System shows the final battle screen.
  • 9.
    Player comes back to the updated map.
Alternative Flows:
  • 6.a.
    If the defence value of the defensive side is at least 50% higher than the attack value of the offensive side then the defensive side automatically wins.
  • 6.b.
    If the attack value of the offensive side is equals to the defence value of the defensive side then System randomly determines the winner.
  • 7.1.
    If the defensive side has lost then System sets the attacked base as destroyed.
    If the Player has won, System changes the target base’s owner from the losing commander to the winning commander.
Extensions:
  • 1.1.
    Player check information about a base.
  • 1.2.
    Player comes back to the map.
